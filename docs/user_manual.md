# 证据标注工具 · 用户手册

面向材料科学论文结构化抽取结果的人工审核平台，支持 Windows 与 macOS 浏览器使用。本手册讲清「界面、操作、状态、指标、导出、快捷键、常见问题」。

---

## 1. 这个工具是做什么的

抽取流水线会从论文里自动抽出结构化字段（论文信息、样品、合金成分、工艺、微观结构、性能……），并为每个字段记录它来自原文的哪一句（evidence 证据）。本工具让你**逐字段核对这些抽取结果**：

- 对的 → **确认**
- 错的 → **修改**成正确值
- 该有却没抽到的 → **补充**
- 拿不准的 → **标记待复核**；与原文矛盾的 → **标记冲突**

最终导出一份高质量、可追溯、并附带 Precision/Recall/F1 的标注结果，供下游使用。每个账号都有独立标注工作区，互不影响。

---

## 2. 界面总览（三栏 + 悬浮窗）

```
顶栏：当前账号 · 论文切换 · 进度 · 保存状态 · 撤销/重做 · 暂存 · 指标 · 任务状态 · 导出 · 帮助 · 退出
┌───────────────┬────────────────────────────┬─────────────────┐
│ 左：证据 / PDF  │ 中：JSON 树（可折叠）         │ 右：字段详情      │
└───────────────┴────────────────────────────┴─────────────────┘
底部：颜色图例                            右下：可拖动悬浮窗（AI助手 / 手册）
```

- **左栏**：两个 Tab。`证据原文`按原文证据块分段展示，点中间字段会自动滚动并高亮对应证据块；`PDF 原文`内嵌论文 PDF 供对照阅读。
- **中栏**：按 schema 结构还原的**可折叠 JSON 树**。相关字段聚在一起（例如 `性能 → mechanical → tensile_properties → yield_strength → value`）。包含值为 `null` 的预定义字段——它们同样需要审核并计入指标。
- **右栏**：选中字段后展示 路径、当前值（可编辑）、原始值对照、证据列表、支持度诊断、备注、状态。

---

## 3. 中间 JSON 树怎么用

- **展开/折叠**：点分组标题左侧的三角，或点标题本身。分组标题上的小色条表示该枝下各审核状态的分布。
- **叶子字段**：每个叶子是一个标注单元，显示 键名、值、证据徽章。点叶子即选中（右栏展开、左侧高亮证据）。
- **行内快捷操作**：叶子行内有 `确认 / 待复核 / 冲突` 按钮，可不打开右栏直接标注。
- **预览 JSON**：每个 bucket（论文级 / 各样品）可一键预览其完整 JSON（含 null、含你已改的值），帮助整体理解。
- **null 字段**：显示为「（空）」。若原文其实有该信息，请把它当作「遗漏」补上（见 §5）。

## 4. 核对与修改字段

1. 点一个字段 → 左栏自动定位并高亮它的证据块。
2. 对照证据判断：
   - 正确 → 点 `✓ 确认`（或按 `c`）。
   - 错误 → 在右栏「当前值」框直接改，回车（或 `⌘/Ctrl+Enter`）保存，状态自动变「已修改」，原值仍保留在导出里。
   - 存疑 → `⚑ 待复核`（`r`）；与原文矛盾 → `✕ 冲突`（`x`）。
3. 一个字段有多条证据时，右栏证据列表可逐条点开，或用 `n`/`N` 在证据块间循环。
4. 证据缺失或定位不到：徽章会显示「缺证据」，右栏会提示；你可在右栏手动填/改证据块 ID。

---

## 5. 补齐遗漏字段（重点）

抽取可能漏掉某些信息。补齐方式**贴合 JSON 结构**：

- 在中间 JSON 树里，把鼠标移到**要补充的那一层**节点（section / 元素 / 分支），会出现一个 `＋`。
- 点 `＋` → 就地弹出小表单，填 `字段名` 与 `值` → 添加。
- 新字段以紫色「已补充」出现在该层级下；点其右侧 `✕` 可删除。
- 也可用中栏底部「＋ 补充字段到该 bucket 根」补在 bucket 顶层。

补充的字段会计入导出与召回（Recall）——因为它代表模型漏抽的内容。

---

## 6. 状态与颜色

颜色只表示**审核状态**（一个维度），证据质量/置信度用**徽章**表示，避免颜色过载。

| 颜色 | 状态 | 含义 |
|---|---|---|
| 灰 | 未处理 | 尚未审核 |
| 绿 | 已确认 | 接受原值 |
| 蓝 | 已修改 | 值被改过 |
| 紫 | 已补充 | 人工新增字段 |
| 琥珀 | 待复核 | 存疑，留待复核 |
| 红 | 冲突 | 与原文矛盾 |

徽章：`证据充分 / 数值匹配 / 弱证据 / 缺证据`、置信度数值、证据块数量、`冲突` 标记。

---

## 7. 保存、任务状态、撤销

- **自动保存**：每次操作后短暂延迟自动落盘，顶栏显示「已保存 / 未保存… / 保存中… / 保存失败」。
- **暂存**：点顶栏「暂存」或按 Windows `Ctrl+S` / Mac `⌘S` 立即存草稿。草稿跨重启保留，随时可继续。
- **任务状态**：`未开始 / 进行中 / 已提交`。首次操作自动转「进行中」；全部核对完可置「已提交」。
- **撤销 / 重做**：Windows `Ctrl+Z` / `Ctrl+Shift+Z` 或 `Ctrl+Y`；Mac `⌘Z` / `⌘Shift+Z`，最多 100 步。

---

## 8. 指标（Precision / Recall / F1）

点顶栏「指标」实时查看；导出包里也有 `evaluation_metrics.json`。

- **golden = 你的人工标注，pred = 原始抽取**，单位是「字段槽位」。
- **TP** = 已确认且非空（模型抽对了）
- **TN** = 已确认且为空（模型正确地没抽，不计入 P/R）
- **FP** = 已修改（改成别的值）+ 冲突（模型抽错了）
- **FN** = 把空字段补成有值 + 已补充字段（模型漏抽了）
- 待复核、未处理**不计入** P/R，但会在 coverage 里报告已审比例，避免「只标了几条却显示高 F1」的误导。
- `Precision = TP/(TP+FP)`，`Recall = TP/(TP+FN)`，`F1` 为二者调和平均；提供总体与分 section 两个粒度。

## 9. 导出

点「导出」后选择导出范围：

- **导出当前论文**：下载 `<paper_id>_annotation_export.zip`，只包含当前打开论文。
- **导出全部分配论文**：下载 `<username>_all_annotation_export.zip`，包含当前账号 `assignments.json` 中已分配的所有论文，并在顶层生成 `ALL_MANIFEST.json`。

每篇论文的目录内包含：

- `field_review.json` ★ 权威扁平结果（每字段：原值/审核值/状态/证据/置信度）
- `evaluation_metrics.json` ★ P/R/F1
- `text_extraction.reviewed.json` 按你的修改回填后的结构化 JSON
- `diff.json` 变更集；`annotation_state.json` 原始标注状态（可回灌继续标）
- `audit_log.jsonl` 操作审计流；`review_summary.md` / `MANIFEST.json` 人工可读汇总

## 10. AI 助手

右下悬浮窗的「助手」标签。可以问：

- **系统操作**：如「怎么补一个遗漏字段？」「冲突和待复核有什么区别？」
- **领域知识**：如「κ-carbide 是什么？」「固溶处理和时效的区别？」
- **标注判断**：把拿不准的字段/值/证据贴给它，帮你判断对错与如何修正。

助手使用 DeepSeek Chat Completions 接口，默认模型为 `deepseek-v4-flash`。在项目根目录 `.env` 中配置 `DEEPSEEK_API_KEY` 后重启服务即可启用在线 AI；也可用环境变量 `ENVIZ_ASSISTANT_API_KEY` 覆盖。未配置或请求失败时，会自动回落到内置 FAQ。每个对话会保留完整多轮上下文；可点「新建对话」开始新会话，也可用下拉框切换历史对话。对话历史按登录账号隔离保存在当前浏览器本地。

悬浮窗可用鼠标、触控板或触摸拖动到任意位置；不需要时可收起。

## 11. 登录与账号隔离

打开工具后先登录。默认本地账号为 `annotator1`、`annotator2`、`annotator3`、`annotator4`，管理员可在部署环境中替换为正式账号。

- 每个账号有独立工作区，默认目录名等于账号名，也可由管理员配置 `workspace`。
- 每个账号的标注文件独立保存在 `data/users/<工作区目录>/annotations/`。
- 每个账号可见的论文由 `data/users/<工作区目录>/assignments.json` 控制，格式为 `{"papers":["paper_id_1","paper_id_2"]}`。
- 同一篇论文给不同账号标注时，进度、修改、补充字段、导出结果互不影响。
- 右下角 AI 助手的历史对话也按账号隔离。
- 退出登录前会尝试保存当前草稿。

## 13. 常见问题

- **改错了怎么办？** Windows 按 `Ctrl+Z`、Mac 按 `⌘Z` 撤销，或把值改回原值（会自动回到未处理），或在右栏点「重置」。
- **null 字段一定要处理吗？** 建议处理：确认「确实没有」（计为 TN），或补成正确值（计为召回）。未处理的会算作 pending，不计入 P/R。
- **证据高亮不到？** 说明该字段的 evidence 定位不到（少数源数据路径不自洽）。可对照 PDF/原文人工核对，并在右栏手动修正证据块 ID。
- **数据会丢吗？** 不会。所有标注写入 `data/users/<工作区目录>/annotations/`，与原始抽取结果隔离；自动保存 + 暂存 + 跨重启保留。
- **在哪里改账号密码？** 推荐在项目根目录 `.env` 里配置 `ENVIZ_USERS_JSON` 覆盖默认账号；格式见 `.env.example`。密码不要写明文，先生成 SHA-256 后填入 `password_sha256`。
- **在哪里分配论文？** 编辑 `data/users/<工作区目录>/assignments.json`。只把该用户应看到的 `paper_id` 放进 `papers` 数组；未出现在数组里的论文，该用户无法在列表、详情、PDF、图片、导出接口中访问。
